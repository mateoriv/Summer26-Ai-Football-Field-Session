# Application Notes

The `app/` package houses the PySide6 desktop client that glues together video playback, CSV-driven metadata, and a virtual field overlay. This document walks through the main window layout and explains what each module contributes.

## Main Window Flow
1. `application.py` instantiates `MainWindow`, wires up the menu bar, and spawns four dock widgets: video player, file browser, data sheet, and virtual field.
2. When you pick a folder (`File → Open Folder`), `fileAccess.load_folder()` scans for CSV files and videos. It autoloads the first match and keeps a tree view pinned to that directory.
3. The custom video player in `video.py` renders frames via OpenCV, overlays YOLO detections, and feeds frame numbers back to the virtual field.
4. Selecting rows in `dataSheet.py` seeks clips and timestamps. The video dock toggles player/yard-marker boxes and updates the progress slider plus current time label.
5. `virtualField.py` consumes per-frame homography data to animate player positions on a stylized field.

## Module Reference
- `application.py` – top-level Qt app; menu actions for opening/remembering folders; saves settings to `~/.hudl_ai_settings.json`; coordinates dock visibility and theme switching.
- `video.py` – custom OpenCV-backed QWidget with frame timer, playback controls, bounding-box overlays, yard-marker toggles, and virtual-field synchronization helpers (`set_current_video_path`, `load_and_set_detection_data`, etc.).
- `fileAccess.py` – builds the file browser dock; exposes `load_folder` and `open_video_file`; generates default CSVs when none exist; manages the tree model for navigating sessions.
- `dataSheet.py` – table model and view for session CSVs; auto-detects video columns, handles row selection to launch clips, and syncs the playback slider/timecode.
- `virtualField.py` – draws a simplified football field, loads homography outputs from `cache/<folder>/homography`, and renders transformed player positions.
- `palette.py` – light/dark palette definitions plus helpers for applying system colors (used by the `Theme` menu).
- `processingDialog.py` – modal dialog that runs single-video processing pipelines, showing log output and progress.
- `batchProcessingDialog.py` – multiprocessing batch runner that executes detection scripts across a folder; includes status updates and error reporting.
- `__init__.py` – empty initializer to mark the directory as a package.
- `requirements.txt` – minimal dependency list if you want to install the UI as a standalone package (`pip install -r app/requirements.txt`).

### Supporting Patterns
- All UI components share state through the `parent` window (e.g., `parent.custom_video`, `parent.current_folder`).
- Cached JSON outputs are expected under `cache/<session>/…`, generated by the scripts in `../scripts/`.
- Menu bar items (`File`, `Window`, `Theme`) are added in `application.py`; on macOS they appear in the global menu bar.

When adding new UI pieces, follow the dock/template pattern used here: keep business logic in helper modules and expose simple methods back to the main window.
